# 文档编辑器手工验收清单

在 `localhost:3001` 后台与博客前台完成以下检查。自动化测试见 `backend/apps/knowledge/tests/test_document_editor_api.py` 与 `frontend` 下 `*.test.ts`。

## 保存与切换

- [ ] Markdown / 富文本 / HTML 三种模式输入后，状态栏显示「待保存 → 保存中 → 已保存」
- [ ] Ctrl/⌘+S 立即保存，不等待 5 秒防抖
- [ ] 切换 MD ↔ 富文本 ↔ HTML 后，刚输入的内容仍在（不丢字）
- [ ] 切换文档或离开页面前，`beforeunload` 在未保存时提示
- [ ] 快速切换「私人笔记 / 发布版」后，各自正文独立、不串内容

## 双正文（raw / published）

- [ ] 草稿：仅编辑私人笔记；发布后博客读 `published_content`
- [ ] 已发布：改私人笔记并保存后，博客**不**自动变（除非点「更新发布」）
- [ ] 「发布版未同步」Tag 在 `raw_content !== published_content` 时出现
- [ ] 「更新发布」将当前笔记同步到发布版；博客刷新后可见
- [ ] 「发布版」分段下编辑并保存，只改 `published_content`
- [ ] 首次「发布」前 flush 最新笔记，发布内容与编辑器一致

## 冲突与错误

- [ ] 双标签同时编辑同一文档：后保存者收到 409 警告，并加载服务器版本
- [ ] 409 后富文本编辑器内容与服务器一致（非仍显示旧本地稿）
- [ ] 网络/权限错误时显示「保存失败」提示（非静默失败）

## 发布与撤回

- [ ] 发布检查弹窗校验标题与即将上线的正文
- [ ] 撤回发布后状态为草稿，博客不可读（若原为公开）

## 斜杠命令（语雀式拼音）

- [ ] 富文本输入 `/` 弹出菜单；输入 `dmk` 匹配「代码块」、`yy` 匹配「引用」、`glk` 匹配「提示色块」
- [ ] `Ctrl/⌘+/` 在任意光标位置插入 `/` 并唤起菜单
- [ ] Markdown 编辑器同样支持 `/` 与拼音筛选；「仅富文本」项灰显且不可选
- [ ] `/mermaid` 或「Mermaid 图表」插入流程图模板

## Mermaid 图表块

- [ ] 富文本 Mermaid 块顶栏：图表/源码/分栏切换、图表类型下拉、缩放 +/-、复制 SVG
- [ ] `Ctrl+Shift+P` 在 Mermaid/PlantUML 块内循环视图模式
- [ ] 分栏模式下源码与预览并排（窄屏纵向堆叠）
- [ ] 阅读端「源码 / 图表」按钮文案与编辑端一致

## 编辑功能

- [ ] Ctrl/⌘+F 查找替换（MD/HTML 与富文本）
- [ ] `Ctrl/⌘+E` 切换行内代码（富文本）
- [ ] 查找替换后关闭页面前仍有未保存提示（若 autosave 未完成）
- [ ] 大纲跳转定位到正确标题行/块
- [ ] `@` 提及插入 `@[title](doc:id)` 语法
- [ ] 粘贴图片上传并写入正文
- [ ] 斜杠命令插入数学、折叠块、Mermaid 等（富文本）
- [ ] F9 专注模式 / Esc 退出

## 博客阅读回归

- [ ] 公开文章详情展示 `published_content`（非私人笔记）
- [ ] HTML 文档 iframe 阅读正常
- [ ] 标签云 / 归档 / RSS 链接仍有效

## 已知限制（非本轮缺陷）

- 富文本 ↔ Markdown 复杂表格、合并单元格可能丢失
- 标题折叠不写入 Markdown，刷新后恢复展开
- 脚注采用简化 `^[...]` 序列化，与标准 GFM 脚注不完全一致
