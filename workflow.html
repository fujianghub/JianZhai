2 本地启动 4 步开机

  1. 启动依赖服务（postgres + redis） 
    $ cd ~/Claude_Code/jianzhai
$ docker compose up -d   # 容器健康后常驻，无需再管
  2. 启动后端 Django 
    $ cd backend
$ source .venv/bin/activate
$ python manage.py runserver 0.0.0.0:8002

    改 Python 文件会 autoreload；改 .env 必须重启进程。

  3. 启动 Celery worker（可选） 
    $ celery -A jianzhai worker -l info

    不启也能跑，但 搜索索引 双向链接 PDF 导出 三项异步不会刷新。

  4. 启动前端（两个模式二选一） 
    $ cd frontend
$ pnpm dev              # 普通 HTTP，localhost 用
$ pnpm dev:https        # 自签 HTTPS，LAN IP 用（消除 Chrome 下载警告）
LAN IP 访问下载提示"不安全"？ 用 pnpm dev:https，且后端 .env 改 JIANZHAI_PUBLIC_ORIGIN=https://<ip>:3001，重启 Django。首次浏览器需点「高级 → 继续访问」一次。

7 故障排查 症状速查

页面打不开

症状原因 → 怎么修
白屏 + Network 500后端崩了。tail -f /tmp/jianzhai-django.log 看 Traceback。
白屏 + Console 报 CORS后端 .env 的 CORS_ALLOWED_ORIGINS 没含当前页 origin。改完重启 Django（autoreload 不读 .env）。
POST / PATCH 403CSRF 失败。CSRF_TRUSTED_ORIGINS 需含当前 origin（含协议）。
Vite 报 502 Bad GatewayDjango 没起。回 §2 重启。

下载与下载警告

Chrome 提示「连接不安全」你在 HTTP LAN IP 上访问。切 pnpm dev:https，浏览器首次「高级 → 继续」一次。
下载文件名是乱码后端 FileResponse(filename=...) 编码问题。检查 apps/exporter/views.py。

Mermaid 渲染失败

红框「Mermaid 渲染失败」看 console 错误。新版 utils/mermaid.ts 已用 JS 颜色解析消 NaN bug；如再现，贴错误给我。
暗主题节点看不清已在 v0.9.4 修：themeVariables.primaryColor 用 nodeFill = mix(surface, accent, 14%)。
全屏按钮无反应检查 previewHtml 是否 ready；hydration 未完成时 fullscreen 会 silently bail。

测试 / CI

本地 pytest 报 No module named 'django'没激活 venv。source .venv/bin/activate。
vitest 跑 DOM 测试报 document is not definedvite.config.ts 用 environment: 'node'。要么用手写 stub（见 api/exports.test.ts），要么加 jsdom。
CI 挂了但本地全过对比 Python / Node 版本；CI 是 fresh 环境，本地有缓存。

其他

改 .env 没生效必须重启 Django。autoreload 不读 .env。
搜索结果不更新Celery worker 没启？手动 python manage.py reindex_search。
双链失效同上，Celery 异步处理。或目标文档已软删除。
AI 端点 503/admin/ai 主开关关了，或 ANTHROPIC_API_KEY 未配。
万能调试入口 · tail -f /tmp/jianzhai-{django,vite}.log 看后端 + 前端实时日志。Chrome DevTools 的 Network + Console 同时开着。
